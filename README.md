# MDP-Based Dynamic Tutorial System in Unity

플레이어의 실시간 스탯 변화에 따라 최적의 성장 경로(마을, 상점, 사냥터, 보스)를 스스로 계산하고 가이드하는 **실시간 동적 튜토리얼 AI 시스템**입니다. 

강화학습의 기초가 되는 **마르코프 결정 과정(Markov Decision Process, MDP)** 알고리즘을 유니티(Unity) 게임 엔진 상에 구현하여, 단순한 하드코딩 방식의 튜토리얼이 아닌 데이터 기반의 유연한 AI 가이드 환경을 구축했습니다.

## 💡 주요 기능 및 핵심 로직 (AI & MDP)

### 1. 벨만 기대 방정식을 통한 가치 연산
실시간으로 변하는 플레이어의 스탯을 바탕으로 각 지역(State)으로 이동(Action)했을 때의 기대 보상을 계산합니다.

$$V^*(s) = \max_a \left[ R(s,a) + \gamma \sum_{s'} P(s'|s,a)V^*(s') \right]$$

### 2. 동적 승률 연산 및 안전 지향 보상 설계
* **동적 승률:** `Clamp(플레이어 공격력 / 타겟 권장 공격력, 0.1, 1.0)` 수식을 적용하여 승률을 실시간으로 계산합니다.
* **보상 함수:** `(경험치 * 승률) - (예상 체력 손실 * 2.0)`을 적용합니다. 
* **페널티:** 사망이 예측되는 무모한 진입에는 `-10,000`의 페널티를, 의미 없는 제자리걸음에는 `-100`의 페널티를 부여하여 최적의 동선을 강제합니다.

### 3. 안정적인 정책 평가 및 개선 (Policy Iteration)
경로 탐색 중 발생할 수 있는 무한 루프를 방지하기 위해 가치 함수가 오차 한계치(`1e-4`) 미만으로 수렴할 때까지 반복 연산하는 **안정화 루프**를 구현했습니다.

## 🛠 시스템 아키텍처 및 최적화 (Architecture & Optimization)

본 프로젝트는 AI 연산의 부하를 줄이고 유지보수성을 높이기 위해 다양한 최적화 기법과 디자인 패턴을 적용했습니다.

### 1. 옵저버 패턴 (Observer Pattern) 활용
* `Update()` 함수의 불필요한 프레임 연산 오버헤드를 제거했습니다.
* C# `Action` 델리게이트를 활용한 **이벤트 주도(Event-Driven) 설계**를 통해, 플레이어의 스탯이 변경되는 시점에만 UI와 AI 로직이 호출되도록 최적화했습니다. 이를 통해 데이터와 뷰의 의존성을 완벽히 분리했습니다.

### 2. 데이터 주도 설계 (Data-Driven Design)
* `ScriptableObject`를 활용해 각 지역(Region)의 데이터와 핵심 로직을 완전히 분리했습니다.
* 프로그래머의 코드 수정 없이, 기획자가 인스펙터 창에서 즉각적으로 난이도 밸런싱을 수행할 수 있는 환경을 구축하여 유지보수성을 극대화했습니다.

### 3. 심리스(Seamless) 맵 트리거 시스템
* 유니티 2D 물리 엔진의 `BoxCollider2D` 트리거를 활용하여, 보이지 않는 경계선을 넘을 때 지역 인덱스를 즉각 갱신합니다.
* 흐름이 끊기는 로딩 화면을 없애 플레이어의 몰입감을 극대화했습니다.

### 4. 통합 전역 매니저 (Global Manager)
* **싱글톤(Singleton)** 패턴과 `DontDestroyOnLoad`를 조합하여 씬(Scene) 생명주기와 독립적인 통합 전역 매니저를 구축했습니다.
* 씬 전환 시 핵심 데이터가 초기화되는 문제를 해결하고, 맵 이동 간에도 AI 연산 데이터가 파괴 없이 보존되도록 설계했습니다.
